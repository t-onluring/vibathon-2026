Phases

4

Each phase builds on the previous one. Current focus is closing Phase 1.5 under the accepted group-level fallback. Phases 2-4 progressively add content extraction capabilities.

Phase 1.5 · Data Model

Parent-Child Sources

Keep the parent-child data model, accept group-level fallback when Telegram forum topics are not publicly parseable, and close the phase with region filtering plus a pluggable health checker baseline.

CURRENT ~1 week 14 files
phase-1.5-parent-child-sources.html →
Phase 2 · Text Pipeline

Event Extraction MVP

Build the first content ingestion pipeline. Extract structured kajian events (ustadz, masjid, waktu, lokasi) from plain text Telegram messages using LLM-based parsing.

NEXT ~2 weeks 9 files $15-75/mo
phase-2-event-extraction-mvp.html →
Phase 3 · Vision

Vision Extraction

Extend the pipeline with vision/OCR to parse image posters — weekly schedules, tabligh akbar flyers, masjid announcements. Covers ~45% of content that text extraction can't handle.

QUEUED ~2 weeks 6 files $60-255/mo
phase-3-vision-extraction.html →
Phase 4 · Production

Full Pipeline

Complete the platform with video extraction, cross-source dedup, near real-time processing, a public API with OpenAPI spec, and community event verification.

QUEUED ~4 weeks 7+ files $100-335/mo
