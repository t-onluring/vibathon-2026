Pipeline Steps

Step 1: Content Type Router

Classify incoming messages as text, image, video, or mixed. Route to appropriate extractor.

content-router.ts classification

Step 2: Image Downloader + Preprocessor

Download images via Telegram Bot API getFile. Resize large images to reduce LLM API costs. Store originals in data/images/.

image-downloader.ts preprocessing

Step 3: Vision Extraction with LLM

Send images to vision-capable LLM with structured extraction prompt. Handle multi-event posters containing 5-7 events per image. Includes few-shot examples for Indonesian Islamic lecture posters.

vision-extractor.ts llm prompt engineering

Step 4: Multi-Event Poster Handling

Weekly schedule posters contain multiple events. Extract each as a separate KajianEvent with shared masjid/address fields.

multi-event-splitter.ts parsing

Step 5: Caption + Image Fusion

Fuse extracted data from both image and caption text. Caption often contains address/location not visible in poster.

caption-fusion.ts data merge

Step 6: Confidence Calibration

Compare vision extraction results against text extraction for messages that have both. Build confidence calibration model.

confidence-calibrator.ts validation

Step 7: Update Ingestion Workflow

Modify GitHub Actions to include image download + vision extraction steps. Add cost monitoring for vision API calls (10-20x more expensive than text).

github actions cost monitoring

Step 8: Dashboard Image Preview

Show poster thumbnail in event card on dashboard. Click to expand full image.

frontend ux
